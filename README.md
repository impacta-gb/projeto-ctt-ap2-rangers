## 💻 Fluxo de Trabalho

Para manter o código organizado e evitar conflitos nos arquivos Markdown (.md), a equipe implementou o seguinte fluxo de desenvolvimento:

1. **Divisão de Módulos:** Os 10 tópicos da linguagem Go foram divididos entre os integrantes para a criação de conteúdo teorico - técnico com exemplos práticos de código.
2. **Desenvolvimento Local:** Cada integrante editou e estruturou seus arquivos correspondentes dentro do diretório `docs/` utilizando o **VS Code** e **Git Bash**.
3. **Sincronização via Git:** O controle foi realizado de maneira contínua pelo terminal, utilizando comandos padrão:
   * `git add .` para palpar as alterações.
   * `git commit -m " descrição do ajuste"` para registrar o progresso.
   * `git push origin main` para enviar as atualizações ao repositório do github.
4. **Ultimos ajustes:** Ao finalizar a estrutura dos tópicos do arquivo Markdown (.md), foi realizado a separação por archivos por de cada topico, assim tambem dando os ultimos detalles
